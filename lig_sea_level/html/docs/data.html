Data table format¶

The table below describes the values expected in a data table.

Inference data table description¶

Parameter

Description

inference_region

(str) name of the region this data belongs to. GIA is solved for in each region indepently.

interglacial

lig for the last interglacial or holocene for the holocene.

sample ID

Sample name.

source

Reference the sample was first reported in.

lon

Longitude (Degrees in decimel form)

lat

Latitude (Degrees in decimel form)

age

Mean age of sample (in kya) assuming normal uncertainty.

age_1sigma

1-\(\sigma\) age uncertainty in ky.

os_age

Open system corrected mean age of sample (in kya) assuming normal uncertainty.

OS_age_1sigma

Open system corrected 1-\(\sigma\) age uncertainty in ky.

custom_age_distribution

none if you want to use normal age uncertainty. Otherwise, a (str) corresponding to the pymc distribution of choice.

age_dist_offset

A value to be added to the custom distribution.

age_dist_scale

A value to be multiply the custom distribution by. Can be used to scale or flip (-1) the distribution.

age_param_1_name

none if the custom distribution has no parameters. Otherwise the name of the parameter you will supply.

age_param_1

Value for parameter 1.

age_param_2_name

none if the custom distribution has either zero or 1 parameters. Otherwise the name of the second parameter you will supply.

age_param_2

Value for parameter 2.

elevation

The elevation of the sample in meters above mean sea level, or the relative sea level of the sample if sample has already been corrected for indicative range.

elevation_1sigma

1-\(\sigma\) age uncertainty in meters.

type

Description indicating how the sample should be treated (not currently used by the code)

comment

Additional comments on the sample (not currently used by the code)

IR_Distribution

none if you do not want to model additional uncertainty associated with the indicate range of the sample. Otherwise, a (str) corresponding to the pymc distribution of choice.

param_1_name

none if the custom distribution has no parameters. Otherwise the name of the parameter you will supply.

Param_1

Value for parameter 1.

param_2_name

none if the custom distribution has either zero or 1 parameters. Otherwise the name of the second parameter you will supply.

Param_2

Value for parameter 2.

emulator_name

Filename containing the neural network emulator weights to use when estimating GIA in this region. The file should be placed in the lig_sea_level/emulator_weights folder.

updown_rate

The mean uplift (positive) or subsidence (negative) rate to apply to the sample in meters per thousand years. Uncertainty is assumed to be normal.

updown_rate_1sigma

1-\(\sigma\) uncertainty in uplift or subsidence rates.
